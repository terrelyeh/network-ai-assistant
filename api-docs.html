📖 怎麼讀這個頁面？

下面是 Swagger UI 自動生成的 API 互動文件。每個 EnGenius Cloud API 都列出來了，可以一邊看 schema 一邊真的試打 — 像 Postman 但在網頁裡。4 步驟操作：

1
先按 Authorize 填 api-key
頁面最上方右側有 🔓 Authorize 按鈕。會跳出兩個欄位：apiKeyAuth（給大部分 skill 用）跟 xAuthToken（給 networks/* 用）— 兩個都填同一個 api-key 值就對了。
2
找你要的 skill
下方分組顯示 13 個 skill 標籤（hvs / networks / org-devices…）。按一下 tag 名稱展開該 skill 底下所有 op。
3
點開 op 看細節 · 行裡兩個 icon
左邊彩色 badge = HTTP method（🟢 GET 讀 · 🟡 POST 寫 · 🔵 PATCH 改 · 🔴 DELETE 刪）。右邊 🔒 = 這個 op 要 auth（已 Authorize 過就 OK）。
4
Try it out → Execute
op 右上角 Try it out → 填 params → Execute → 看真實回應、curl 命令。⚠ TBD 開頭的 op（在 troubleshoot skills 下）打不通 — 等 RD 補 dolphin URL。

💡 Server dropdown（在 Authorize 旁邊）只改「Execute 送到哪個 base URL」，不改 op 清單本身。staging / dev / prod 切換就在這。
🔍 Filter 搜尋框（tag 清單上方）即時過濾 tag + op — 輸入「license」只剩 license 相關、輸入「troubleshoot」只剩 troubleshoot 系列。94 個 op 太多時用這個快很多。

⚠ 關於 47 個 x-rd-pending ops — RPC / Subscribe op 還沒拿到 HTTP method/path 文件，所以暫時放在 /x-rd-pending/<skill>/<op> pseudo-path 下，UI 上看到 ⚠ TBD 前綴。RD 補完後重跑 scripts/build-openapi.py 自動補完。詳見 rd-meeting/06-api-doc-questions.md。
