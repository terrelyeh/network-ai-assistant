網路裡的兩種人
都該有適合自己的 AI 體驗

員工不該為了 Wi-Fi 慢去 call IT，SMB IT 也不該因為不會設儀表板就放棄監控 — Network AI-Assistant 用一套後端、兩種模式（員工自助 + IT 助理）同時解這兩件事。

兩種體驗 · 兩種人

這個產品同時解決兩種人的痛點

每個 persona 都有自己的「痛 → AI 介入 → 結果」故事

MODE A · 員工模式
👤
員工 / 終端使用者
非技術人員 · 公司一般員工 / SMB 老闆
😫 痛點
「會議室 Wi-Fi 又卡了，但去 call IT 又怕被嫌、要排隊。先忍著吧。」
✨ AI 體驗
→打開 chat 用人話問：「會議室 Wi-Fi 很慢」
→AI 反問：「現在剩幾格訊號？」
→AI 給立即可做的小動作（換位置 / 重連）
→解不了 → AI 自動開單給 IT
🎯 結果
80% 問題自助解決，不用煩 IT、不用等回應
風險等級 🟢 極低（純讀取）
MODE B · SMB IT 模式
🛠
SMB IT
缺經驗的 IT · 一個人扛全公司網路
😫 痛點
「Grafana 我設一次就放棄了 — 不知該看什麼指標，又不會調 query。出問題還是只能 Google。」
⭐ 殺手功能
📊 Dashboard Builder
用講的就生出儀表板 — 不需懂 query / 不需知 metric / 不需會排版
✨ AI 體驗
→Anomaly Panel 主動告訴：「會議室 AP 異常」
→AI 跑 playbook 給診斷 + 建議（含證據）
→看到「降 TX power 6dB / 影響 12 client」 → 一鍵核准
→10 分鐘後 AI 確認修好（沒效自動 rollback）
🎯 結果
從「調查員」變「複核者」， 30 分鐘人工診斷壓到 30 秒
風險等級 🟡 受控（需核准）
