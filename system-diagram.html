系統有哪些 block · 它們怎麼接

沒有 ring、沒有 primitive 細節，但比 PM/MKT 概覽多了「實際 component」 — 給技術型業務、PM 排 roadmap、與 partner 對接時用。

BLOCK ②
Applications · 4 個入口
Chat 互動診斷、Anomaly Panel 主動通知（★ 新增）、 Dashboard Builder 自然語言視圖、Custom Skills 自訂能力。 同一個 AI 引擎服務不同情境。
BLOCK ③
AI Engine · 系統大腦
三大職責：Skill Routing（選對 playbook）、 Safety + Approval（攔下危險動作要人核准）、 Reasoning + Audit（每個決定有完整推理鏈可查）。
BLOCK ④
Skills Library · 能力庫
分三類：Diagnostic（被動，問才查）、 Monitoring（主動，24×7 跑）、 Custom（社群 / 客戶自訂）。所有 playbook 都用同一個 YAML 格式。
BLOCK ⑤
MCP Tool Layer · 標準介接
Anthropic 業界標準協定。把 device API 包裝成 LLM 可呼叫的工具。 最大價值：客戶自家 AI 也能用同一個 MCP 接進來 — 不被 vendor lock-in。
SIDE
📦 GitHub Skills Library
Playbook 的版本控制 + 社群分享。EnGenius 官方 + 社群貢獻共用同一格式。 客戶 IT / 顧問可以發布、訂閱 playbook — 生態系飛輪的開端。
SIDE
🤖 Customer AI Engine
客戶自家的 AI（Microsoft Copilot、自建 LLM 等）可透過同一個 MCP server 連進來查網路、跑 playbook。 消除企業客戶 #1 的 vendor lock-in 焦慮。
