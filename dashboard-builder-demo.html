🛠 HOW IT WORKS

從一句話到儀表板 · 4 個步驟

01

解析意圖

LLM 把自然語言「我想看 X」解析成「要查什麼資料、哪些指標、什麼時間範圍」。

02

選 widget

從 widget library（限定一組）選最適合的視覺化：KPI、line chart、heatmap、table 等。LLM 不從零畫圖，只組合既有元件。

03

產生查詢

把意圖轉成具體的 telemetry query（透過 primitive 呼叫）。LLM 不直接寫 SQL，由 backend 做 query 翻譯，避免錯誤。

04

排版 + 儲存

AI 用 grid layout engine 自動排版，使用者預覽、確認、儲存。後續可以「再加一個...」refine。

🆚 vs 傳統 DASHBOARD 工具

為什麼這對 SMB IT 是質變

面向 傳統工具 (Grafana / PowerBI / Tableau) Dashboard Builder
建立方式 手動拖拉、選 metric、選圖表、設參數 用講的，AI 自動組合
知識門檻 需懂 query 語法（PromQL/SQL） 不需要任何技術知識
知道該看什麼 使用者要自己知道 AI 主動建議該包含什麼
視覺設計 使用者自己排版 AI 自動 grid 排版
修改 / refine 每次都要進編輯模式 「再加一個 X」就好
學習曲線 幾小時 ~ 幾天 第一次就會用
對 SMB IT 友善度 「設一次就放棄」 每天都會打開用
