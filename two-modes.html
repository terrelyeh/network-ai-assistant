PRODUCT POSITIONING · 兩模式定位

Network AI-Assistant — 一套後端，兩種模式

同一個 AI 引擎，依使用者角色提供不同 UI 與能力範圍
🏠 主入口 🎯 產品入口 🛠 工程入口
同一個 AI 後端 · 兩種使用者體驗

不是「給工程師的 AI」
也不是「給員工的 AI」 — 是兩種人都能用的 AI

員工自己問問題、SMB IT 引導式診斷 — 同一套後端，UI 隨身分變化。
每種模式都有合適的能力範圍與風險等級，避免一刀切。

MODE A · 員工模式
👤
Employee Chat
「員工自助 Wi-Fi 客服」
非技術用戶不需要 call IT

核心能力

  • 💬純自然語言對話
  • 🎓教育式回應（解釋名詞）
  • 📍個人視角（我的會議室、我的位置）
  • 🎫解不了 → 自動開 ticket 給 IT
  • 📊個人簡易 dashboard
範例對話
「我家會議室 Wi-Fi 很慢」
了解。您現在用的是 7F 會議室嗎？方便看一下手機訊號剩幾格？
「2 格」
先試試靠近門口看訊號有沒有變強。如果還是 2 格我幫您開單給 IT 好嗎？
寫入動作 ❌ 不會
風險等級 🟢 極低
主要動作 Read + Escalate
MODE B · SMB IT 模式
🛠
SMB IT Cockpit
「缺經驗的 IT 也能管好網路」
AI 是您的資深工程師助理

核心能力

  • 🧠診斷 + 引導式修復
  • 📊Dashboard Builder (跨模式 wedge · 看 Demo →)
  • 🔔Anomaly Panel 主動通知
  • ✋Ring 0/1 動作 + 一鍵核准
  • ↩️自動驗證 + 沒效自動 rollback
  • 🎓每個建議都附完整 reasoning
範例對話 · Dashboard Builder
「我想看這週網路有沒有異常」
建您一個「本週健康」儀表板：anomaly 趨勢、客戶數變化、Top 5 問題設備、各 site 健康分數。要建立嗎？
「先預覽 → OK」
已儲存 ✓ 之後從首頁可以直接看。要再加什麼面板隨時跟我說。
寫入動作 ⚠ Ring 0 + 1
風險等級 🟡 受控
主要動作 Diagnose + Approve
