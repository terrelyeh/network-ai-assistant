🎯 Phase 1 · 三個必做格子

這三格之中只要做兩個，產品就能上市；做完三個就是完整 v1.0

A③
員工自助 chat 客服
TAM 最大、風險最低、競品最弱。每個員工都會用 → 入門就能 viral。風險低（純讀為主）→ 不用扛 ring system 全套。
B② ⭐
Dashboard Builder（殺手）
SMB IT 最痛的地方。不會設儀表板 / 不知該看什麼 → 直接消除「Grafana 學一次就放棄」的摩擦。這是 demo 一次就 wow 的功能。
B③
SMB IT 診斷 + 引導修復
Roger 原始設計核心。把 30 分鐘人工診斷壓到 30 秒 + 一鍵修復。完整跑過 ring + approval + closed-loop 的標竿案例。

建議的 Phase Roadmap

把 8 個格子排成 2 個季度

PHASE 1
Validate
M0–M3 · MVP
目標：證明兩模式的核心命題成立 — 員工會用、SMB IT 愛用、能省下時間。
策略：全部 read-mostly + 受控 Ring 1，避開高風險動作。
A② 個人 dashboard A③ 員工自助 chat B② Dashboard Builder ⭐ B③ SMB IT 診斷
PHASE 2
Expand
M3–M6 · GA · 拓展功能
目標：拓展到管理 / 部署維度，把整套 SMB IT 流程做完整。
策略：開放更多 Ring 1 動作 + 引導式部署，仍嚴格守 approval。
B① 引導式部署 B④ 「我想看 X」式探索 B③+ 更多 playbook（VPN / IPv6 / SD-WAN） A 補強 · 員工 chat 多語系

📌 deliberately 不做 MSP / multi-tenant / 跨組織 console — 那是不同 ICP，需要 multi-tenant、billing、white-label。 把「公司內 IT」做透再說。架構保留 mode-agnostic，未來真要擴後端不會大改。
