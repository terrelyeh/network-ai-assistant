2 Modes vs Wedge Product · 對照

為什麼 wedge 跟 modes 不同軸

Modes 是依「誰用」分（A 員工 / B SMB IT）。 Wedge 是「賣什麼」 — Dashboard Builder 跨兩個模式都能用。

面向 Mode A · 員工 Mode B · SMB IT ⭐ Dashboard Builder
(跨模式 wedge)
主要用戶 非技術員工 缺經驗 IT 所有 user（跨模式）
軸性質 依「誰用」 依「誰用」 依「賣什麼」
UI 形態 單一 chat 視窗 多 panel cockpit 4 步驟 wizard
主要互動 對話 查看 + chat + 核准 描述 → 確認 → 用
AI 角色 客服 / 教練 副駕駛 「翻譯員」
寫入動作 無 (只開單) Ring 0/1 + approval Ring 0/1 (save)
商業角色 附贈 / 包含於 suite 主訂閱 / Suite 核心 ⭐ Wedge / Land 工具
競爭 frame 內建客服 vs Mist / Aruba（紅海） vs Grafana（藍海）
Demo 焦點 「不用 call IT」 「30 秒看完當日狀況」 「用講的就有 dashboard」

💡 展示順序建議（依 wedge 策略）

Phase 1 land → 主推 Dashboard Builder（最容易 demo · 最低風險 · TAM 最大）。
對 PM/MKT 講故事 → Dashboard Builder（wow）→ Mode A（員工自助降 IT 負擔）→ Mode B（IT 主控台）。
對技術買家 → Mode B（cockpit + 核准流程 + ring 安全機制）。
展會 demo → 30 秒 Dashboard Builder wow → Mode A → Mode B 升級劇情。
Expand 階段 → 客戶上 Dashboard Builder 後，upsell 到 diagnose / monitor / approval 完整 suite。
