🛠 HOW IT WORKS

從一句話到儀表板 · 4 個步驟

01

解析意圖

LLM 把自然語言「我想看 X」解析成「要查什麼資料、哪些指標、什麼時間範圍」。

02

選 widget

從 widget library（限定一組）選最適合的視覺化：KPI、line chart、heatmap、table 等。LLM 不從零畫圖，只組合既有元件。

03

產生查詢

把意圖轉成具體的 telemetry query（透過 primitive 呼叫）。LLM 不直接寫 SQL，由 backend 做 query 翻譯，避免錯誤。

04

排版 + 儲存

AI 用 grid layout engine 自動排版，使用者預覽、確認、儲存。後續可以「再加一個...」refine。

🆚 vs 傳統 DASHBOARD 工具

為什麼這對 SMB IT 是質變

面向 既有預設 dashboard Dashboard Builder
內容範圍 受限於開發者預先排好的幾個 用講的就生任意客製版本
新角度需求 提需求 → 排 roadmap → 等下一版 秒生 — IT 自己跟 AI 說就有
知道該看什麼 只能挑現有 dashboard AI 主動建議該包含什麼指標
跨資料組合 通常一個 dashboard 一個維度 AI 跨 site / device / 時段組合
修改 / refine 要找到設定介面、手動拖拉 「再加一個 X」就好
學習曲線 每加一個自訂視圖都要重摸 第一次就會用
對 SMB IT 友善度 能力受預設限制 每天都能問新問題、生新 dashboard
